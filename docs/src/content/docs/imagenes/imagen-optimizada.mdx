## Descripcion

Este endpoint sirve imagenes almacenadas en el bucket y permite reescalarlas o convertirlas. Si no envias parametros de optimizacion, la imagen se devuelve tal como esta guardada.

## Parametros de ruta

| Parametro | Tipo | Descripcion |
| --- | --- | --- |
| `imgFilename` | string | Nombre del archivo tal como esta en el storage. |

## Parametros de consulta

| Parametro | Tipo | Descripcion |
| --- | --- | --- |
| `width` | number | Ancho objetivo en pixeles. |
| `height` | number | Alto objetivo en pixeles. |
| `format` | string | `webp`, `jpeg`, `png`, `avif`. |
| `quality` | number | Calidad de compresion (1-100). |

## Ejemplos de uso

```bash
curl -L "https://tu-dominio/api/image/solo-leveling-cover.webp" \
  --output cover.webp
```

```bash
curl -L "https://tu-dominio/api/image/solo-leveling-cover.webp?width=320&height=480&format=jpeg&quality=75" \
  --output cover-320x480.jpg
```

## Respuesta

La respuesta es binaria (imagen). El servidor define el `Content-Type` segun el parametro `format`:

- `webp` (por defecto si no envias `format`).
- `jpeg`, `png`, `avif` si se solicitan.

Tambien se aplica cache agresiva para assets estaticos:

- `Cache-Control: public, max-age=31536000, immutable`.

## Errores comunes

- `404`: la imagen no existe en el storage.
- `500`: fallo al optimizar la imagen con los parametros enviados.
