## Descripción

Obtiene un arreglo con los episodios mas recientes precalculados. La lista se devuelve ordenada de mas nuevo a mas antiguo.

El campo `image` es una URL de proxy (`/api/image/:token`) que redirige a una URL firmada temporal.

## Respuesta

Cada elemento del arreglo contiene:

| Campo | Tipo | Descripción |
| --- | --- | --- |
| `originalLink` | string | URL del episodio en el proveedor original. |
| `image` | string \| null | URL de proxy firmado para miniaturas. |
| `episode` | number | Número de episodio detectado. |
| `title` | string | Título del episodio. |
| `episodeId` | string | Identificador único, útil para otras rutas. |
| `animeId` | string | Identificador del anime relacionado. |

## Ejemplo de uso

```bash
curl -X GET "https://tu-dominio/api/episodes/latest?limit=20"
```

## Ejemplo de respuesta

```json
[
  {
    "originalLink": "https://www3.animeflv.net/ver/solo-leveling-15",
    "image": "https://tu-dominio/api/image/ZXhhbXBsZS1lcGlzb2RlLXRva2Vu",
    "episode": 15,
    "title": "Solo Leveling - Episodio 15",
    "episodeId": "solo-leveling-15",
    "animeId": "solo-leveling"
  }
]
```

## Casos comunes

- Para obtener las fuentes de un episodio, usa `episodeId` en `/episodes/:episodeId/sources`.
- Para el detalle, usa `episodeId` en `/episodes/:episodeId`.

> Nota: Si necesitas compatibilidad con la ruta antigua de fuentes, aun existe el alias `/episodes/sources/:id` pero esta deprecado.
