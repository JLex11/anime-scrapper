## Descripción

Devuelve un arreglo con la información resumida del catálogo. El origen se pagina de manera secuencial y la página inicial es `1`.

## Parámetros de consulta

| Parámetro | Tipo | Descripción |
| --- | --- | --- |
| `page` | number | Página solicitada (por defecto `1`). |

## Respuesta

Cada elemento contiene:

| Campo | Tipo | Descripción |
| --- | --- | --- |
| `animeId` | string | Identificador único del anime. |
| `title` | string | Título principal. |
| `type` | string \| null | Tipo de anime (TV, OVA, Película, etc.). |
| `rank` | number \| null | Ranking numérico si está disponible. |
| `otherTitles` | string[] \| null | Títulos alternativos. |
| `description` | string \| null | Sinopsis. |
| `originalLink` | string \| null | URL original en el proveedor. |
| `status` | string \| null | Estado de emisión. |
| `genres` | string[] \| null | Géneros detectados. |
| `images` | object \| null | Imágenes optimizadas para el frontend. |
| `relatedAnimes` | array \| null | Animes relacionados cuando están disponibles. |
| `created_at` | string | Fecha de inserción ISO 8601. |
| `updated_at` | string | Fecha de última actualización ISO 8601. |

## Ejemplo de uso

```bash
curl -X GET "https://tu-dominio/api/animes?page=2"
```

La respuesta es un arreglo de animes que incluye atributos listos para renderizar tarjetas en el frontend.

## Notas

- Los campos opcionales pueden llegar como `null` si el proveedor no publica esa información.
- `images.coverImage` y `images.carouselImages[].link` apuntan al endpoint `/api/image` con parámetros opcionales de optimización.
- Si la fuente cambia, algunos valores (como `rank` o `status`) pueden variar entre consultas.
